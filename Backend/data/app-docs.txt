## 1. Overview
This project is a MERN stack e-commerce platform.
It provides a smooth user experience for browsing products, viewing product details, reading reviews, and placing orders.
The products are cakes, bouquets, plants, chocolates, combos including plants, chocolates, dry fruits.
Price range of proucts is from Rs. 500 to 2000
cakes of flavours - chocolate, vanila, mango
bouquets of roses, jarbera, lily
chocolates hampers of dairy milk, kitkat, ferrerro rocher
plants - money plant, Aglaonem, jade, 

The project supports:
- Product categorisation
- Product detail pages with reviews
- User authentication (login/register)
- Shopping cart & checkout
- Order management
- Responsive UI design

## 2. Technology Stack
**Frontend:**
- React.js (Functional components + Hooks)
- React Router (for navigation)
- Axios (for API requests)
- CSS Flexbox + Media Queries (for responsiveness)

**Backend:**
- Node.js + Express.js (API server)
- MongoDB + Mongoose (Database)
- bcrypt.js (password hashing)
- JSON Web Token (JWT) for authentication

**Other:**
- Cookies for storing recently viewed products
- Placeholder for chatbot integration

## 4. Features

### 4.1 Product Browsing
- Homepage with category display (cakes, bouquets, plants, chocolates, combos)
- Each category links to `/products/:category`
- Product cards display image, name, price, and rating

### 4.2 Product Details
- Clicking on a product shows details (`/product/:id`)
- Shows:
  - Product image
  - Name & description
  - Price
  - Reviews (user, comment, rating, createdAt)

### 4.3 Reviews
- Users can leave reviews (rating + comment)
- Backend stores reviews in `product.reviews` array
- Frontend displays reviews with user and date

### 4.4 Cart & Checkout
- Add products to cart
- View cart, update quantities, remove items
- Checkout process with address and payment simulation

### 4.5 Recently Viewed Products
- Products viewed are stored in browser cookies
- Displayed in a "Recently Viewed" section on homepage
- Filtered from full product list using `_id` matches

### 4.6 Responsive Design
- Flexbox layout with wrapping
- Images adjust size on smaller screens
- Full width display for small devices

## 5. API Endpoints

### **Product Routes**
- `GET /api/products` → Get all products
- `GET /api/products/:id` → Get single product by ID
- `POST /api/products/:id/reviews` → Add review to a product

### **User Routes**
- `POST /api/users/login` → Login user
- `POST /api/users/register` → Register user
- `GET /api/users/profile` → Get user profile (auth required)

### **Order Routes**
- `POST /api/orders` → Create new order
- `GET /api/orders/:id` → Get order details
